# Reporting QA / UAT Validation Pack (SQL)

## Objective
Validate reporting outputs after data pipeline changes or migrations to ensure dashboards and reports align with expected business rules. This project demonstrates a repeatable QA/UAT workflow including test cases, validation queries, issue logging, and stakeholder-ready summaries.

## What This Pack Includes
- QA/UAT test case checklist (expected vs actual)
- SQL validation scripts to verify metrics and logic
- Report output validation framework (pre/post release)
- Defect logging template and sign-off summary

## Skills Demonstrated
Reporting QA, UAT Support, Business Rule Validation, SQL Validation (JOINs, CTEs), Data Reconciliation, Root Cause Analysis, Documentation
